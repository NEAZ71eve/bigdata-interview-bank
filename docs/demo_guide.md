# 现场演示脚本

> 面试中如需现场演示项目，按以下流程操作。全程约 3-4 分钟，每步配话术。

---

## 演示前准备（面试前 5 分钟）

```bash
# 确保环境就绪
cd bigdata-interview-bank
python interview_bank/harness_engine.py  # 预跑一次确认能跑通
```

打开浏览器标签页：
1. GitHub 仓库首页：https://github.com/NEAZ71eve/bigdata-interview-bank
2. 本地 IDE 打开项目目录

---

## Step 1：展示 GitHub 仓库（30 秒）

**操作**：切换到浏览器 GitHub 标签页

**话术**：
> 这是我的项目仓库，根目录 README 重点介绍了多 Agent 协作流水线和 Harness 量化评测体系。`agents/` 目录下有 5 个 Agent 的完整定义，`interview_bank/` 是题库核心，`docs/` 是面试准备材料。

---

## Step 2：展示 Agent 定义（40 秒）

**操作**：在 GitHub 点击 `agents/` → 打开 `generator.md`

**话术**：
> 每个 Agent 有完整的 prompt 定义。比如这个 generator 是智能出题 Agent，它的职责是变式出题——有场景变式、角度变式、深度变式、反向变式四种类型，还负责派生代码题和难度分级。prompt 里明确规定了输入文件、输出格式、质量约束和回传机制。

**关键展示点**：翻到"职责1：变式题生成"部分，展示四种变式类型的具体例子。

---

## Step 3：运行 Harness 评测引擎（60 秒，核心高潮）

**操作**：切换到终端

```bash
cd interview_bank
python harness_engine.py
```

**话术**（边运行边说）：
> 现在演示 Harness 量化评测引擎。它会对题库做四维评测——

（等输出出来后）

> 可以看到四维全部 PASS：
> - **H1 重复率**：6216 对比对，高相似对 0 个
> - **H2 答案准确性**：参数密度每题 7.4 个，远超阈值 2.0
> - **H3 覆盖率**：9 大技术栈全覆盖，无缺失
> - **H4 难度均衡**：L1 到 L4 偏差全在 ±7% 以内

> 退出码是 0，可以直接接入 CI/CD pipeline。

---

## Step 4：展示质检报告（30 秒）

**操作**：打开 `interview_bank/harness_report.md`

**话术**：
> 引擎自动生成这份质检报告，含汇总表、各维度指标、修复建议。最有价值的是这个——

**关键展示点**：翻到 README 里的"修复前后对比表"

> 首次评测 L1 只有 3 道、偏差 -12% 直接 FAIL，我补了 12 道题后重测，偏差收敛到 -1.6% PASS。这就是 Harness 闭环的价值——不是写个脚本就完事，是"检测→FAIL→修复→重测→PASS"的完整闭环。

---

## Step 5：展示题库内容（30 秒）

**操作**：打开 `interview_bank/phase3_expanded_bank.md`，搜 `[L4]`

**话术**：
> 题库有 112 道题，这是其中一道 L4 专家级题目——

**关键展示点**：挑一道 L4 题（如 SAQ-011 Exactly-Once），展示题干 + 标签 + 关联题

> 每道题都有难度标签、技术栈标签、知识图谱关联。再看答案——

**操作**：打开 `phase4_saq_answers.md`，展示对应答案

> 答案按"原理+流程+参数+对比表+追问+生产踩坑"六件套输出，这是 L3 以上题的标配。

---

## Step 6：收尾（15 秒）

**话术**：
> 整个项目 5 个 Agent 零人工干预，54 分钟跑完全流水线。代码和 Agent 定义都开源，Harness 引擎零依赖可直接跑。您有什么想深入看的吗？

---

## 演示节奏控制

| 步骤 | 时间 | 重要性 | 如果时间紧可跳过 |
|------|------|--------|----------------|
| Step 1 GitHub 仓库 | 30s | 中 | 可跳，直接 Step 3 |
| Step 2 Agent 定义 | 40s | 中 | 可跳 |
| Step 3 运行 Harness | 60s | **高** | **不可跳** |
| Step 4 质检报告 | 30s | **高** | 可合并到 Step 3 |
| Step 5 题库内容 | 30s | 低 | 可跳 |
| Step 6 收尾 | 15s | - | - |

> **铁律**：如果只有 1 分钟，只做 Step 3（运行 Harness）+ Step 4（展示修复前后对比）。这是最能体现技术深度的部分。
