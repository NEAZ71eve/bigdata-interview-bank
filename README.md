# 大数据面试题库 · bigdata-interview-bank

> **5 个 AI Agent 协作跑完五阶段流水线，全自动产出 112 道题 + 74 道标准答案，Harness 四维量化质检 ALL PASS。**
>
> 本项目的核心价值不在于题库本身，而在于**多 Agent 协作流水线的工作方法论**与**Harness 量化评测体系**——前者证明了 AI Agent 团队可以零人工干预完成复杂内容生产，后者让"题库质量"从主观感受变为可量化、可迭代的客观指标。

## 一、多 Agent 协作工作流程（核心）

### 1.1 架构设计理念

传统 AI 出题是"一个模型一把梭"，质量不可控、过程不可追溯。本项目采用**职责拆分式多 Agent 流水线**——将题库生产拆解为采集、清洗、出题、阅卷、质检五个独立阶段，每个阶段由专职 Agent 负责，**串行联动 + 阶段内并行**，全程零人工干预。

核心设计原则：
- **单一职责**：每个 Agent 只做一件事，做到极致
- **串行解耦**：前序阶段产出是后续阶段输入，阶段间通过文件传递
- **并行加速**：无依赖的阶段并行执行（如简答阅卷 + 代码阅卷同时进行）
- **主理人编排**：Team Lead 负责建队、调度、汇总，不代写成员产出
- **消息中转**：所有跨成员信息流经主理人，成员不互相直连

### 1.2 五大 Agent 角色定义

| 阶段 | Agent | 角色名 | 核心职责 | 输入 | 输出 |
|------|-------|--------|---------|------|------|
| Phase 1 | **collector** | 采集员 | 定向爬取牛客网大数据面经，WebSearch 补充，过滤无关岗位，提取题目原文/面试场景/高频公司 | 面经报告 + 搜索关键词 | `phase1_raw_interviews.md`（40 道原始题） |
| Phase 2 | **cleaner** | 清洗员 | 原始题去重、分类（选择/简答/代码）、清洗冗余、设计选择题选项、统一格式、标注初始难度 | Phase 1 产出 | `phase2_standardized_questions.md`（40 道标准化题） |
| Phase 3 | **generator** | 出题员 | 基于知识图谱生成变式题（角度/场景/反向设问）、派生手撕代码题、四级难度标注、关联考点标签 | Phase 2 产出 | `phase3_expanded_bank.md`（100 道扩展题） |
| Phase 4a | **saq-grader** | 简答阅卷员 | 50 道简答题纠错、标准化重构、剔除过时知识点，以"原理+流程+优缺点+场景+话术"模板输出 | Phase 3 简答题部分 | `phase4_saq_answers.md`（50 道标准答案） |
| Phase 4b | **code-grader** | 代码阅卷员 | 20 道代码题落地实现、逻辑校验、中文注释、复杂度分析、测试用例 | Phase 3 代码题部分 | `phase4_code_solutions.md`（20 道参考实现） |
| Phase 5 | **harness** | 评测引擎 | 四维量化评测（重复率/准确性/覆盖率/均衡度），PASS/FAIL 判定，输出修复清单 | 全量题库 + 答案 | `harness_report.md`（质检报告） |

> Phase 4a 与 4b **并行执行**，互不依赖，由主理人同时调度。

### 1.4 协作机制详解

**阶段间通信**：Agent 之间不直接通信，所有信息流经主理人中转。每个 Agent 完成任务后，通过 `SendMessage` 将产出回传主理人，主理人核验后传递给下一阶段。

**调度方式**：
- Phase 1-3：严格串行，前序完成才启动后续
- Phase 4a/4b：主理人同时下发任务，两个阅卷 Agent 并行工作
- Phase 5：主理人汇编全部产出后，运行 Harness 引擎做最终质检

**异常处理**：Harness 质检发现 L1 题严重不足（3 道 vs 目标 15 道），主理人立即启动补救——补 12 道 L1 题 + 重新质检，直到四维全部 PASS。

**闭环验证**：每个阶段产出都经过主理人核验（文件大小、行数、题目数量、标签一致性），不合格不下发下一阶段。

## 二、Harness 量化评测结果（核心）

### 2.1 评测体系设计

Harness 引擎是本项目的核心亮点——它让题库质量**可量化、可复现、可迭代**。引擎对题库全量内容做四维量化评测，每维有明确阈值，输出 PASS/FAIL 判定 + 修复清单，退出码 0/1 可接入 CI/CD。

### 2.2 四维评测结果（最新）

运行 `python interview_bank/harness_engine.py` 实测结果：

| 维度 | 评测内容 | 阈值 | 实测值 | 判定 |
|------|---------|------|--------|------|
| **H1 题目重复率** | 题干 Jaccard 相似度全量两两比对（6216 对） | 高相似对占比 ≤ 5% | **0 重复**（89% 题对相似度 < 0.3） | ✅ PASS |
| **H2 答案准确性** | 真实参数/API 标注密度 + 对比表格 + 复杂度覆盖率 | 参数密度 ≥ 2/题 | **7.4/题**（398 次命中 / 54 道简答） | ✅ PASS |
| **H3 考点覆盖率** | 9 大技术栈分布 + 缺失/薄弱检测 | 无缺失栈 | **全覆盖**（Kafka 19 / Spark 20 / Flink 17 / Hive 16...） | ✅ PASS |

## 五、快速开始

### 运行 Harness 质检
```bash
cd interview_bank
python harness_engine.py
# 输出四维评测结果 + 生成 harness_report.md
# 退出码 0 = ALL PASS，1 = 存在 FAIL 项
```

### 使用场景
- **日常刷题**：按 `phase3_expanded_bank.md` 顺序，L1→L2→L3→L4
- **模拟面试**：选择题 30 + 简答 10（L2-L3）+ 代码 2（1 L3 + 1 L4）
- **查漏补缺**：按技术栈标签检索薄弱项
- **面试官出题**：选择题直接作为笔试题，简答题截取分点作为评分参考

## 六、核心创新点

1. **多 Agent 分工协同流水线**：五大角色解耦协作，从原始素材到成品题库全自动落地，替代人工整理
2. **Harness 量化质量体系**：自研四维评测引擎，题库质量从主观感受变为可量化、可迭代的客观指标
3. **闭环修复机制**：质检发现偏差自动触发修复（L1 从 3 道→15 道），直到 ALL PASS
4. **变式出题突破瓶颈**：基于真实高频考点衍生新题，解决传统题库原题枯竭问题
5. **全自动纠错汰旧**：阅卷 Agent 修正网传错误答案、淘汰过时技术知识点
